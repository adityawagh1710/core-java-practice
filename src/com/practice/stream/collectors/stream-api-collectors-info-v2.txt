Java Collectors (Stream API)
==============================================================

Core Idea
─────────
• Collectors are used mainly with stream .collect(Collector)
• They define how to accumulate/mutate/combine stream elements into a final result
• Most come from static factory methods in java.util.stream.Collectors
• Very common in production code and interviews

Static Import Tip (makes code cleaner)
import static java.util.stream.Collectors.*;

1. Basic Collectors – to collections
==============================================================
toList()             → List<T> (mutable ArrayList by default)
toSet()              → Set<T>  (HashSet)
toUnmodifiableList() → List<T> immutable (Java 10+)
toUnmodifiableSet()  → Set<T> immutable
toCollection(Supplier) → custom collection e.g. LinkedList::new, PriorityQueue::new

Example:
List<String> names = List.of("Anna", "Bob", "Charlie", "David");

List<String> list   = names.stream().collect(toList());
Set<String>  set    = names.stream().collect(toSet());
List<String> unmod  = names.stream().collect(toUnmodifiableList());

2. toMap – single key/value
==============================================================
toMap(keyMapper, valueMapper)
toMap(keyMapper, valueMapper, mergeFunction)   ← handles duplicate keys
toMap(keyMapper, valueMapper, mergeFunction, mapSupplier)

Common patterns:
Map<String, Integer> nameToLength = names.stream()
    .collect(toMap(
        identity(),           // key = the element itself
        String::length
    ));

// Duplicate key example (keep first / last / sum / etc.)
Map<String, Employee> byId = employees.stream()
    .collect(toMap(
        Employee::getId,
        identity(),
        (old, neu) -> old       // keep old on conflict
        // or (old, neu) -> neu
        // or (old, neu) -> old.getSalary() + neu.getSalary()
    ));

3. groupingBy – the #1 interview collector
==============================================================
groupingBy(classifier)                            → Map<K, List<T>>
groupingBy(classifier, downstream)                → custom aggregation

Most frequent downstream collectors:
  counting()           → Map<K, Long>
  summingInt/Long/Double(ToInt/Long/DoubleFunction)
  averagingInt/Long/Double
  maxBy/minBy(Comparator)
  reducing(initial, accumulator) / reducing(accumulator)
  joining() / joining(delimiter) / joining(del, prefix, suffix)
  mapping(mapper, downstream)   ← transform before aggregating

Examples:

// Group employees by department
Map<String, List<Employee>> byDept = employees.stream()
    .collect(groupingBy(Employee::getDepartment));

// Count per department
Map<String, Long> countByDept = employees.stream()
    .collect(groupingBy(Employee::getDepartment, counting()));

// Average salary per department
Map<String, Double> avgSalary = employees.stream()
    .collect(groupingBy(
        Employee::getDepartment,
        averagingDouble(Employee::getSalary)
    ));

// Top earner per department
Map<String, Optional<Employee>> topByDept = employees.stream()
    .collect(groupingBy(
        Employee::getDepartment,
        maxBy(Comparator.comparingDouble(Employee::getSalary))
    ));

// Multi-level grouping (department → city → list)
Map<String, Map<String, List<Employee>>> byDeptAndCity = employees.stream()
    .collect(groupingBy(
        Employee::getDepartment,
        groupingBy(Employee::getCity)
    ));

4. partitioningBy – special case of groupingBy (boolean key)
==============================================================
partitioningBy(Predicate)                  → Map<Boolean, List<T>>
partitioningBy(Predicate, downstream)

Example:
Map<Boolean, List<Integer>> evenOdd = numbers.stream()
    .collect(partitioningBy(n -> n % 2 == 0));

// With downstream
Map<Boolean, Long> countEvenOdd = numbers.stream()
    .collect(partitioningBy(n -> n % 2 == 0, counting()));

5. Other very useful collectors
==============================================================
joining() / joining(", ") / joining(", ", "[", "]")
counting()                  → long
summingInt/toLong/toDouble
summarizingInt/Long/Double  → Int/Long/DoubleSummaryStatistics (min,max,avg,count,sum)

Example – statistics:
DoubleSummaryStatistics stats = employees.stream()
    .collect(summarizingDouble(Employee::getSalary));

System.out.println(stats.getAverage());
System.out.println(stats.getMax());

6. Custom collector (rare but asked in senior interviews)
==============================================================
Collector<T, A, R> of(
    Supplier<A> supplier,
    BiConsumer<A,T> accumulator,
    BinaryOperator<A> combiner,
    Function<A,R> finisher,   // optional
    Collector.Characteristics... characteristics
)

Characteristics: CONCURRENT, UNORDERED, IDENTITY_FINISH

Quick Code Snippets – Favorite Interview One-liners

// Frequency map
Map<String, Long> freq = words.stream()
    .collect(groupingBy(identity(), counting()));

// Sorted frequency descending
List<Map.Entry<String, Long>> topWords = words.stream()
    .collect(groupingBy(identity(), counting()))
    .entrySet().stream()
    .sorted(Map.Entry.<String,Long>comparingByValue().reversed())
    .limit(10)
    .toList();

// Join with commas
String csv = names.stream().collect(joining(", ", "Names: ", "."));

Best Practices 2026
───────────────────
• Use static import Collectors.* to keep code clean
• Prefer groupingBy + downstream over collect + forEach
• Avoid side-effects in collectors
• Be careful with parallel streams + non-concurrent maps
• Use toUnmodifiableList/Set when result shouldn't change
